Feasibility Study 

Technical Considerations 

FlatMate is a web-based application designed to help flatmates manage shared household responsibilities such as groceries, expenses, and pantry inventory and for flat notices. The system will consist of a frontend interface for users to interact with, a backend server for handling application logic, and a database to store information related to users, expenses, grocery lists, and inventory items. The technology stack that will be used includes JavaScript with React for the frontend, Node.js with Express for the backend, and PostgreSQL for the database, with deployment planned on a cloud-based platform such as Render or AWS. These technologies are commonly used and suitable for scalable web applications such as FlatMate. We already have experience working with the above tools which will reduce the learning needed to use this technology stack. Overall, FlatMate is technically feasible as it can be implemented using existing technological tools.   

 

Operational Considerations 

FlatMate is designed to address common challenges faced by people living in shared flats. These challenges include keeping track of shared groceries, dividing costs fairly, remembering who owes money, and managing pantry items that are shared between flatmates. It can be used as a centralised platform where flatmates can see and update essential shared information. Users will be able to add groceries not in the inventory that needs to be purchased, keep records of the weekly or monthly expenses such as rent or utilities and keep track of the items in the inventory. The system will be used to calculate fair splitting of expenses depending on how the flat discusses the split to be on. This reduces confusion, miscommunication, and keeps transparency between the flat, and can also be a point of reference if a disagreement comes up. FlatMate will be designed to be simple and intuitive, making sure that the users wouldnt need alot of technical training or necessarily be tech-savy to be able to use it. It will also be run on common web browsers like Chrome, Edge and FireFox so users wont need to do any extra software installations to be able to use it. Given how the system aligns with current real-life issues faced with flatting, it is operationally feasible.
